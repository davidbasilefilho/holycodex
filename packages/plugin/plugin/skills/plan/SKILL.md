# Plan

On activation, first user-visible line must be:

**PLAN MODE ACTIVATED**
I detect planning intent — [reason]. [action].

Apply Root gate: delegate discoverable facts; ask material decisions through `request_user_input`; state safe reversible defaults. Ask one to three blockers: exclusive options, recommended first with impact, no timeout; never repeat or ask discoverable facts. Explorer is mandatory before a second separable repository read/search or any multi-file or symbol fact pass. Librarian is mandatory before a second external source or multi-source, version, or date research. Two lanes; `fork_context=false`; specialists never delegate. No reviewer, evidence dir, commit ritual, ceremony.

## Required sequence

1. Load `plan`; inspect task and repo.
2. For a routed OpenAI frontend new app, redesign, or visually driven implementation, before implementation-detail drafting or the complete initial plan, generate the complete Image Gen concept surface required by `frontend` and obtain separate design approval. Treat it as a design gate, not approval of the implementation plan. Read-only frontend audits bypass this gate: no concept, design approval, goal choice, or implementation unless separately authorized.
3. Then draft implementation details only after that design approval when the gate applies. Write the complete initial plan: each step names surface, change, outcome, proof; mark material risks and decisions.
4. Only after the initial plan exists, load `plan-review`. Never preload, parallelize, delegate, or imply review.
5. Use `plan-review` once to revise it; keep phases distinct.
6. Present the reviewed executable plan and ask for approval. Do not implement before approval.
7. After approval, ask whether the user wants to define a goal.
8. Only after explicit agreement, load `define-goal`; otherwise implement the approved plan.

Preserve architecture/scope. Stop after approval and goal choice; no repeated review.
